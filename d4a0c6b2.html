安装git

Windows

在使用Git前我们需要先安装 Git。Git 目前支持 Linux/Unix、Solaris、Mac和 Windows 平台上运行。Git 安装包下载地址为：http://git-scm.com/downloads下载好，按默认安装即可（推荐安转在D盘中）。安装完成后，在开始菜单里找到“Git”->“Git Bash”，双击蹦出一个类似命令行窗口的东西

在开始栏里面出现这个就说明安装成功

linux安装命令

apt-git install git

通过指令安装git，如果报错如下

root@pe7e1154dd684d1e907eb5cb96bca29a-task0-0:/code# git init bash: git：未找到命令

则需要先更新软件安装包

apt-get update

运行结果如下:

然后运行以下指令：

apt-get install git

初始化设置用户名和邮箱

git config --global user.name"your name"

git config --global user.email email@mail.com

git config --global credential.helper store

创建仓库

init：初始化当前目录为仓库，初始化后会自动将当前仓库设置为master

创建一个新的本地仓库（省去project-name则在当前目录创建）

git init or git init <project-name>

下载一个远程仓库

git clone <url>

git介绍

传输文件到远程仓库

整体步骤如下：

git add . 将工作区的所有文件上传到暂存区

git add file.txt 将单个文件file.txt文件添加到暂存区域

git commit -m” 文件名” 将暂存区的文件上传到本地仓库

git remote add origin https://openi.pcl.ac.cn/dongliang/model1

git push –set-upstream origin master

git pull origin master

Git操作日志可以看出，以下是发生的情况以及每一步改正的思路：

  1. git pull 操作时出现 MERGE_HEAD：这是因为你在之前的操作中启动了合并过程，但没有完成它。Git提示你需要先提交合并或放弃它。你之后使用了 git commit 来提交解决冲突并完成合并。

    git merge --abort 或者 git rebase --abort #放弃未完成的合并操作

  2. 配置全局用户名和邮箱：Git提醒你全局用户名和邮箱地址未配置，这是为了确保每次提交都包含正确的作者信息。你使用 `git 

    config` 命令配置了全局用户名和邮箱地址。

git config --global user.name "Your Name" git config --global user.email you@example.com #配置全局的用户名和邮箱地址

  3. git pull 操作时再次出现 MERGE_HEAD：在此之后，你再次运行 git pull 时，又遇到了未完成的合并。这可能是因为之前的合并操作没有正确完成或提交。

    git commit -m "解决冲突并完成合并" #提交已解决冲突的更改

  4. git pull 操作时等待编辑器关闭：Git尝试打开一个文本编辑器以等待你输入合并提交消息。然而，由于你的环境中没有正确配置编辑器，出现了错误。这时你可以选择提交消息编辑或者使用 --no-edit 选项来跳过这一步。

    git pull --no-edit 或者 git pull --rebase --no-edit
#跳过提交消息编辑，完成合并

  5. git push 遭到拒绝：在尝试将更改推送到远程仓库时，你遇到了非快进（non-fast-forward）推送的问题，这是因为本地和远程分支的历史记录不一致。解决方法是先拉取远程更改，然后再推送。

    git pull origin master或者 git pull --rebase origin master
 # 拉取远程更改并合并到本地分支，这将确保你的本地分支与远程分支同步

  6. git pull origin master：运行这个命令拉取了远程更改并解决了合并问题。你之前的提交解决了冲突并完成了合并。

    git push --set-upstream origin master #推送更改到远程仓库

  7. git push 操作成功：最后，你成功地将更改推送到远程仓库。分支已更新并设置为跟踪远程分支。

启智平台调用模型，克隆命令

bash

bash是什么？

  • bash是一个命令处理器, 运行在文本窗口中, 并能执行用户直接输入的命令.
  • bash还能从文件中读取Linux命令, 称之为脚本.
  • bash支持通配符, 管道, 命令替换, 条件判断等逻辑控制语句
